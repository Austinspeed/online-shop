**Online Shop** is a full-featured e-commerce REST API that gives you everything you need to build a shopping experience from the ground up. With a simple HTTP interface, you can register and manage user accounts, browse a live product catalog, maintain a per-user shopping cart, and submit orders — all without touching a UI. The API returns standard JSON responses and uses session-based authentication via cookies, making it easy to integrate with any HTTP client or web framework.

## How it works

The Online Shop API follows a straightforward flow that mirrors a real shopping experience:

1. **Register** — Call `POST /api/auth/signup` with your email, password, name, and shipping address to create a new account.
2. **Log in** — Authenticate with `POST /api/auth/login`. The server sets a secure session cookie that is automatically attached to all future requests.
3. **Browse** — Use `GET /api/products` to explore available products. This endpoint is public and requires no session.
4. **Add to cart** — Send `POST /api/cart/items` with a `productId` to add a product to your cart. Retrieve your cart at any time with `GET /api/cart`.
5. **Place an order** — When you're ready to check out, submit your order through the orders endpoint. Your cart is cleared and an order record is created.

<Note>
  Authentication is required for all cart and order operations. Requests to endpoints such as `GET /api/cart`, `POST /api/cart/items`, and `POST /api/orders` without a valid session will return a **401 Unauthorized** response. Make sure to log in and include your session cookie before calling these endpoints.
</Note>
